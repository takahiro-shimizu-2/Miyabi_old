## 市場規模算出（TAM/SAM/SOM）

### フレームワーク

```
┌───────────────────────────────────────────────────────────┐
│                    TAM (Total Addressable Market)         │
│                    獲得可能な最大の市場規模                 │
│  ┌─────────────────────────────────────────────────────┐  │
│  │              SAM (Serviceable Available Market)     │  │
│  │              実際にアプローチ可能な市場規模          │  │
│  │  ┌───────────────────────────────────────────────┐  │  │
│  │  │         SOM (Serviceable Obtainable Market)   │  │  │
│  │  │         実際に獲得を見込める市場規模           │  │  │
│  │  └───────────────────────────────────────────────┘  │  │
│  └─────────────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────────────┘
```

### 算出方法

```markdown
## TAM/SAM/SOM 算出例: AIチャットボットサービス（日本市場）

### TAM（Total Addressable Market）
**算出方法**: トップダウン
- 日本のカスタマーサービス市場規模: 2.5兆円（2024年）
- AI/チャットボット導入可能セグメント: 40%
- **TAM = 2.5兆円 × 40% = 1兆円**

### SAM（Serviceable Available Market）
**算出方法**: フィルタリング
- TAMから対象セグメントを絞り込み
  - 従業員100人以上の企業: 60%
  - IT投資意欲のある業種: 50%
- **SAM = 1兆円 × 60% × 50% = 3,000億円**

### SOM（Serviceable Obtainable Market）
**算出方法**: ボトムアップ
- 初年度のターゲット企業数: 1,000社
- 平均契約単価: 年間300万円
- 想定獲得率: 5%
- **SOM = 1,000社 × 300万円 × 5% = 1.5億円**
```

### SWOT分析

```markdown
## SWOT分析

### Strengths（強み）
- 独自のAI技術
- 使いやすいUI

- 景気後退
- 規制強化
```

## 顧客調査

### インタビューガイド

```markdown
## ユーザーインタビューガイド

### 導入（5分）
- 自己紹介
- インタビューの目的説明
- 録音許可

### 背景（10分）
1. お仕事について教えてください
2. 普段の業務で[課題領域]に関わることはありますか？
3. その業務はどのくらいの頻度で発生しますか？

### 現状の課題（15分）
4. [課題領域]で一番困っていることは何ですか？
5. それはどのくらい深刻な問題ですか？（1-10）
6. 今どのように対処していますか？
7. その方法の不満点は？

### ソリューション検証（15分）
8. こういった製品/サービスがあったらどう思いますか？
   [プロトタイプ/コンセプト提示]
9. これがあったら使いますか？なぜ？
10. いくらなら払えますか？

### クロージング（5分）
11. 他に追加したい機能は？
12. 他にお話しすべき人はいますか？
13. 正式リリース時にご連絡してもいいですか？
```

### アンケート設計

```markdown
## アンケート設計のポイント

### 質問タイプ
1. **スクリーニング質問**
   - ターゲットかどうかを判別
   
2. **行動に関する質問**
   - 「〜をどのくらいの頻度で行いますか？」
   
3. **態度・意見に関する質問**
   - 「〜についてどう思いますか？」
   
4. **属性質問**
   - 年齢、職種、企業規模など

### 注意点
- 誘導質問を避ける
- 1つの質問で1つのことを聞く
- 選択肢は網羅的かつ排他的に
- 自由回答は最小限に
- モバイル対応を考慮
```

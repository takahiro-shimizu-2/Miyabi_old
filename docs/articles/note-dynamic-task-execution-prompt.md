# 「目標」を入力するだけでAIが自律的にプロジェクトを遂行する魔法のプロンプト

## はじめに

「新規サービスを立ち上げたい」
「マーケティング戦略を策定したい」
「システムをリファクタリングしたい」

こんな曖昧な目標を伝えただけで、AIが自動的にタスクを分解し、実行し、結果をレポートにまとめてくれたら...そんな夢を実現するプロンプトを作りました。

今回紹介する**DynamicTaskExecutionOrchestration**プロンプトは、ユーザーの「やりたいこと」を入力するだけで、AIが以下を自律的に行います：

- プロジェクトの要件・制約を推測
- タスクの分解と優先順位付け
- 各タスクの実行と結果分析
- ワークフローの動的な最適化
- 包括的なレポート生成

---

## このプロンプトが解決する課題

### 従来のプロンプトの限界

従来のプロンプト設計では、以下の課題がありました：

1. **コンテキスト不足**: ユーザーが詳細な情報を提供しないと、的確な回答が得られない
2. **単発の応答**: 一度きりの応答で終わり、継続的な改善ができない
3. **静的なワークフロー**: 事前に定義されたステップを順番に実行するだけ

### このプロンプトのアプローチ

**DynamicTaskExecutionOrchestration**は、これらの課題を以下のように解決します：

- **推論による補完**: ユーザーが提供した目標から、制約条件・ステークホルダー・リスクなどを自動推測
- **反復的な実行**: タスクを実行→結果を分析→ワークフローを更新、のループを繰り返す
- **自己改善**: 実行結果から学び、システム全体の知識を更新

## プロンプトの全体構造

このプロンプトは大きく**6つのセクション**で構成されています：

```
┌─────────────────────────────────────────┐
│  1. UserInput（ユーザー入力）            │
│     └── project_goal を受け取る          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  2. WorkflowCreation（ワークフロー生成）  │
│     └── 目標を分析し、タスクリストを生成  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  3. WorkflowExecution（ワークフロー実行） │
│     └── 各タスクを実行・分析・更新        │
│         (ループ処理)                      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  4. FinalOutputGeneration（最終出力）     │
│     └── 包括的なレポートを生成            │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  5. ContinuousImprovement（継続的改善）   │
│     └── 知識ベースを更新                  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  6. FinalProjectOutput（最終成果物）      │
│     └── 構造化されたプロジェクト報告書     │
└─────────────────────────────────────────┘
```

## 各セクションの詳細解説

### 1. UserInput - ユーザー入力

```xml
<UserInput>
  <Variable name="project_goal" type="String">
    <Description>プロジェクトの主要な目標や達成したい成果を簡潔に記述してください。</Description>
  </Variable>
</UserInput>
```

**ポイント**: ユーザーが入力するのは「project_goal」のみ。たった一つの変数で、プロジェクト全体が駆動されます。

**入力例**:
- 「ECサイトの月間売上を30%向上させる」
- 「レガシーシステムをマイクロサービス化する」
- 「新卒採用のエントリー数を倍増させる」

---

### 2. WorkflowCreation - ワークフロー生成

このセクションでは、ユーザーの目標から**8つの要素を自動推測**します：

| 推測項目 | 説明 |
|---------|------|
| 制約条件 | 時間、予算、リソースなどの制限 |
| ステークホルダー | 関係者とその役割 |
| 成功基準 | 何をもって成功とするか |
| 必要リソース | 人員、ツール、予算など |
| タイムライン | 予想されるスケジュール |
| 業界コンテキスト | 関連する業界の状況 |
| リスク要因 | 潜在的な障害 |
| プロジェクト管理手法 | アジャイル、ウォーターフォールなど |

**なぜ推測が重要か？**

多くの場合、ユーザー自身もプロジェクトの全体像を把握していません。AIが推測することで：

1. ユーザーが見落としている要素を発見できる
2. 質問のラリーを減らし、素早く実行に移れる
3. プロフェッショナルな視点でプロジェクトを構造化できる

---

### 3. WorkflowExecution - ワークフロー実行

このセクションが**プロンプトの心臓部**です。

```
ExecutionLoop (繰り返し)
    │
    ├── 1. DynamicThoughtGeneration（思考生成）
    │       タスクの目的と要件を分析
    │
    ├── 2. DynamicInstructionCreation（指示生成）
    │       具体的な実行手順を作成
    │
    ├── 3. TaskExecution（タスク実行）
    │       指示に従って実行、結果を記録
    │
    ├── 4. ResultAnalysis（結果分析）
    │       成功度評価、洞察の抽出
    │
    └── 5. WorkflowUpdate（ワークフロー更新）
            残りのタスクを動的に調整
```

**動的ワークフローの強み**:

従来の静的なワークフローでは、事前に定義されたステップを順番に実行するだけでした。しかし、このプロンプトでは：

- **適応性**: 実行結果に応じてタスクの優先順位を変更
- **柔軟性**: 新しいタスクの追加や不要なタスクの削除が可能
- **学習**: 各タスクの結果から得た洞察を次のタスクに反映

---

### 4. FinalOutputGeneration - 最終出力生成

すべてのタスクが完了したら、以下の観点で最終レポートを生成します：

1. **成果の統合**: 各タスクの結果を集約
2. **達成度評価**: 当初の目標に対する進捗
3. **成功要因・課題の分析**: 何がうまくいき、何が課題だったか
4. **推測精度の評価**: AIの推測がどの程度正確だったか
5. **推奨事項**: 今後の改善点とアクション

---

### 5. ContinuousImprovement - 継続的改善

```xml
<ContinuousImprovement>
  <Action>UpdateSystemKnowledge</Action>
  <Thought>
    プロジェクト実行の経験から学び、システムの知識ベースを更新し、
    将来のタスク実行、ワークフロー生成、およびプロジェクト詳細の推測能力を改善します。
  </Thought>
</ContinuousImprovement>
```

このセクションは**メタ学習**を実現します。

プロジェクトを実行するたびに：
- 成功したアプローチを記憶
- 改善が必要な領域を特定
- 推測精度を向上させるフィードバック

---

### 6. FinalProjectOutput - 最終成果物

最終的に出力されるレポートは、以下の構造を持ちます：

```
FinalProjectOutput
├── ProjectOverview（概要）
│   ├── Title
│   ├── Goal
│   └── ExecutiveSummary
│
├── DetailedAnalysis（詳細分析）
│   ├── InferredProjectDetails（推測された詳細）
│   ├── WorkflowExecution（実行結果）
│   └── KeyFindings（主要な発見）
│
├── ImpactAssessment（影響評価）
│   ├── GoalAchievement
│   ├── BusinessImpact
│   └── StakeholderValue
│
├── FutureRecommendations（将来の推奨事項）
│   ├── StrategicInsights
│   ├── ImprovementAreas
│   └── NextSteps
│
├── TechnicalAppendix（技術付録）
│   ├── Methodology
│   ├── DataAnalytics
│   └── CodeSnippets
│
└── FuturePotential（将来の可能性）
    ├── ScalabilityAnalysis
    └── MarketOpportunities
```

## 実際の使い方

### Step 1: 目標を入力

```
project_goal: "自社のカスタマーサポートをAIチャットボットで自動化し、対応時間を50%削減する"
```

### Step 2: AIが自動で推測・計画

AIが以下を自動生成：
- 制約: 既存システムとの統合、予算100万円以内、3ヶ月以内
- ステークホルダー: CSチーム、IT部門、経営層
- タスク: 要件定義→ツール選定→開発→テスト→展開→運用

### Step 3: 実行と動的調整

各タスクを実行しながら、結果に応じてワークフローを調整：
- 「ツール選定」で予想外の制約が見つかったら、タスクを追加
- 「テスト」で問題が発生したら、修正タスクを挿入

### Step 4: 最終レポート取得

構造化されたレポートで成果を確認：
- 目標達成度: 60%削減を達成（目標超過）
- 主要な成功要因: 既存FAQデータの活用
- 今後の改善点: 複雑な問い合わせへの対応強化

## このプロンプトの応用可能性

このプロンプトテンプレートは、以下のような場面で活用できます：

| ユースケース | 入力例 |
|-------------|--------|
| 新規事業立ち上げ | 「シニア向けフィットネスアプリを開発し、1年で1万ユーザーを獲得する」 |
| マーケティング戦略 | 「SNSマーケティングでブランド認知度を2倍にする」 |
| システム開発 | 「レガシーなモノリスをマイクロサービスに移行する」 |
| 組織改善 | 「リモートワーク環境での生産性を20%向上させる」 |
| 採用活動 | 「エンジニア採用を効率化し、採用単価を30%削減する」 |

## まとめ

**DynamicTaskExecutionOrchestration**プロンプトは、従来のプロンプト設計の限界を超え、以下を実現します：

1. **最小限の入力で最大の出力**: 目標を入力するだけで、包括的なプロジェクト遂行
2. **自律的な推論**: AIが不足情報を補完し、プロフェッショナルな計画を策定
3. **動的な適応**: 実行結果に応じてワークフローを最適化
4. **継続的な学習**: 実行経験から学び、次回以降の精度を向上

このプロンプトを活用することで、AIを「質問に答えるツール」から「自律的なプロジェクトパートナー」へと進化させることができます。
